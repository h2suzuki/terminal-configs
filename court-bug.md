# CAVEAT: Court bug

## 現象

- Claude Code 2.1.148 以降 / Opus 4.7 以降で "court" とうい文字列が混入し Tool Call が失敗するバグが頻発
- モデルには、全ての Tool Call が成功したように見えるが、実際には全て失敗する。 コード生成・バグ修正・動作確認・テストが成功したような幻覚を見ることになるため、全ての作業が信頼できなくなる
- 実際には何も実行できていないので、後から終わった作業の成果物を見つけられず、混乱してトークンを余計に消費する迷路に迷い込む
- モデル自身が自ら気づくことは非常に困難。court / count といった単語を見かけても、ちょっとした書き損じと軽視して、気にしないで先に進んでしまう

## 緩和策

- 英語での思考・発話に限定する。 CJK で頻繁に発生するため
- Tool 発行前の prose をなるべく短くする
- Fable 5 をつかう（Fable 5 では発生が確認されていない）

## 発生後の回復手段

- セッションレジュームすると新しいセッションにも伝染するので、直ちにセッションリセットする以外、救済の道はない
- Handoff の Write/Edit Tool も静かに失敗していることが多く、直前のどの Tool Call まで成功しているかも確認不可能なので、記録は発話するに留め、transcript log を経由したフレッシュセッションへの引き継ぎが最も確実
